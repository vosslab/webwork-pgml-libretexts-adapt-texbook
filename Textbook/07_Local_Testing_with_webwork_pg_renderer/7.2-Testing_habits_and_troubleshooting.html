Most failures in WeBWorK problem authoring are predictable: a blank is not tied to the intended answer, a random value produces an impossible case, or the student-facing render hides an assumption you thought was obvious. This section collects a small QA pass that catches most issues quickly.

Variant sweeps (catch broken randomization)

  • Render multiple variants and scan for impossible or meaningless cases.
  • Watch for awkward rounding, repeating decimals, or values that make the story unclear.
  • Constrain random values so the biology stays plausible (concentrations, probabilities, rates).

Boundary values (catch grading edge cases)

  • Test values near expected boundaries (zero, one, and simple fractions).
  • Check that tolerances and rounding expectations match what the prompt asks for.
  • For text answers, decide what variants you accept (plural/singular, hyphens, and case).

Student-view behavior (catch workflow surprises)

  • Confirm the prompt reads cleanly and does not rely on hidden formatting.
  • Confirm each blank appears where you expect and has clear unit and format cues.
  • Use logs and any available health check to separate PG syntax problems from environment or rendering issues.

Apply it today

  • Adopt a repeatable QA pass: preview, sweep variants, test one boundary case, then publish.
  • When something breaks, keep the failing seed and file path so you can reproduce the same case.
