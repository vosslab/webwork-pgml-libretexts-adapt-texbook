Local testing can save a lot of time when you are iterating on PG and PGML: you make an edit, render the problem, and see the result immediately without importing into a course or publishing an assignment.

This chapter introduces webwork-pg-renderer as a lightweight way to preview and test problems locally. The goal is not to replace full testing in WeBWorK or ADAPT, but to give you a tight edit-preview loop for day-to-day authoring.

What local rendering is good for

  • Quick checks while you are editing a prompt, table, or multi-part blank structure.
  • Reproducing a failure with a fixed seed and file path, so you can debug the same variant.
  • Spot-checking that substitutions, formatting, and answer blanks render as you expect.

What it does not replace

  • Student-view behavior inside a real course (permissions, settings, and submit controls).
  • ADAPT import and assignment workflow checks.
  • Final review of grading behavior across multiple variants in the real deployment.

Apply it today

  • Pick one problem you are editing and use local rendering to preview the prompt after each small change.
  • When a variant breaks, keep the failing seed so you can reproduce the same case while debugging.
