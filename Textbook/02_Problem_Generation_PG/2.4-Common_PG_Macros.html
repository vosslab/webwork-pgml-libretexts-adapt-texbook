This page only covers the PG you need to support a PGML prompt.

Every PGML problem still needs a short PG preamble that loads the macros providing PGML, MathObjects, and course defaults. This page lists the small set of macro files you will use most often when writing PGML-first problems.

Macros you almost always load

  • PGstandard.pl: core PG helpers and defaults.
  • MathObjects.pl: MathObjects and robust answer checking.
  • PGML.pl: PGML markup (BEGIN_PGML, answer blanks, formatting).
  • PGcourse.pl: course-level helpers and sensible defaults.

Common optional macros

  • parserRadioButtons.pl: radio-button answers (multiple choice with one correct).
  • parserPopUp.pl / parserCheckbox.pl: popup or checkbox style answers.
  • answerHints.pl: targeted feedback for common wrong answers (use sparingly).

Advanced macro topics you will see referenced

Some macro families show up often in existing WeBWorK problem banks. You do not need them for most PGML-first life science prompts, but it helps to recognize the names when reading or adapting older problems.

Recognition only: this section is here so you can read older problems, not so you feel required to use these macro families.

  • Algorithmic questions: patterns and helpers for controlled randomization, so each student gets a different (but equivalent) version.
  • Inexact values (significant figures): the InexactValue macro and related helpers for sig-fig-aware grading and presentation. Use only when your course truly requires strict sig-fig handling; otherwise, prefer explicit tolerances and clear reporting expectations.

Minimal PGML-first loadMacros block

DOCUMENT();

loadMacros(
	"PGstandard.pl",
	"MathObjects.pl",
	"PGML.pl",
	"PGcourse.pl",
);

Rule of thumb

  • Start with the minimal macro set above.
  • Add one extra macro only when you need a specific interaction type.
  • Keep the setup short; most authoring should happen in PGML.
