This page only covers the PG you need to support a PGML prompt.

PG is the language used to define WeBWorK problems. It is implemented in Perl and uses macros to provide features like answer checking, randomization, and formatting.

This guide discourages writing long prompts in "regular PG". Instead, treat PG as the setup layer and write the student-facing prompt in PGML.

The minimum PG you need

  • DOCUMENT(); at the top of the file.
  • loadMacros(...); including PGML.pl and MathObjects.
  • A small setup block: random values, computed answers, and answer checkers.
  • A PGML text block between BEGIN_PGML and END_PGML.
  • ENDDOCUMENT(); at the end of the file.

This page does not cover custom Perl subroutines, long loops, or heavy conditional prompt generation. If you feel pulled toward writing a Perl novel, stop and consider tightening ranges, simplifying the prompt, or choosing a different interaction pattern instead.

How much Perl do I need to know?

Usually: a little. Most problems only need a few scalar variables (like $a, $b) and a few MathObjects created with Compute() or Real(). If you keep your setup small and your prompt in PGML, you can author effectively without learning advanced Perl patterns.

A mindset for science problems

  • Randomize the values, not the story.
  • Keep units explicit and consistent (and prevent unit drift: μM vs mM, μL vs mL).
  • Use realistic ranges (no negative concentrations, no 10,000 μL pipettes).
  • For text answers, state naming expectations (gene/protein symbols, case, and hyphens).
  • Prefer one or two answer blanks with clear interpretation.
  • When you need a table or structured layout, reach for PGML helpers rather than ad-hoc HTML.

Safe defaults (micro-pattern)

  • Choose a context early.
  • Define a small set of values with constrained ranges.
  • Define answers as MathObjects.
  • Attach answers directly to blanks in PGML.
