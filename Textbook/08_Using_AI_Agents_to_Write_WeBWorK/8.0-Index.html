AI coding agents can write WeBWorK PG problem files for you, so you can focus on your subject expertise instead of learning Perl syntax. Think of it like directing a trained lab technician: you describe the experiment you want, and the technician carries out the procedure. You still need to know what a good result looks like, but you do not need to pipette every reagent yourself.

Since late 2025, several AI coding agents have become capable enough to produce working WeBWorK problems from a plain-language description. This chapter walks you through choosing an agent, giving it the right reference material, connecting it to a local renderer so it can test its own output, and refining your prompts to get better problems with less back-and-forth.

Start here

  • If you have never used an AI coding agent, start with What are AI Agents? to understand what they are and pick one that fits your setup.
  • If your agent writes code that does not render correctly, read Knowledge Documents for AI Agents to give it better reference material, then Connecting AI Agents to the PG Renderer so it can test its own output before you review it.
  • If your agent already produces working problems but you want better results, go to Advance AI Agents Advice for prompt-engineering techniques and refinement strategies.

Prerequisites

  • A running PG renderer on your local machine (see Setting Up the PG Renderer).
  • Basic understanding of WeBWorK problem structure (see Problem Generation (PG)).
  • An AI coding agent installed on your computer (see What are AI Agents? for options and installation links).
