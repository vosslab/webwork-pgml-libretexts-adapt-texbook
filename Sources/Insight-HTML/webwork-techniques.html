The WeBWorK system, originally developed by the Mathematical Association of America and the National Science Foundation, is an open-source online homework platform widely used in mathematics and related disciplines. Within the ADAPT homework system, WeBWorK serves as one of the integrated assessment engines, providing algorithmically generated problem sets with randomized values that ensure each student receives a unique version of a problem. This promotes academic integrity while allowing unlimited attempts with immediate feedback, supporting mastery-based learning. By embedding WeBWorK into ADAPT, instructors can draw from an extensive library of thousands of vetted math and STEM problems, customize assignments, and seamlessly deliver them alongside other interactive question types, giving educators and students a unified and flexible homework experience.

WeBWorK problems can be authored using two related coding systems: PG (Problem Generation) and PGML (Problem Generation Markup Language). PG is the original language, built on Perl with custom macros that handle randomization, answer checking, and math rendering. It is highly flexible and powerful but can be difficult to read and write, since it mixes Perl code, LaTeX, and formatting commands. To address this, PGML was developed as a more user-friendly markup layer on top of PG. PGML uses a Markdown-like syntax where authors can write text, include LaTeX math, and insert answer blanks or multiple-choice questions with simple, clean commands.

PGML is the preferred language to build WebWork questions in ADAPT. It is a simplified markup language to write and format math problems in a clean, readable way. It combines plain text with LaTeX-style math for equations, along with special commands for answer blanks, multiple-choice questions, and other interactive features. Unlike raw Perl coding in WeBWorK, PGML makes problem authoring more approachable by handling much of the underlying formatting automatically. For example, PGML automatically aligns equations, inserts answer boxes, and checks student input, allowing instructors to focus on problem content rather than complex code. It is now the preferred method for writing new WeBWorK problems.
