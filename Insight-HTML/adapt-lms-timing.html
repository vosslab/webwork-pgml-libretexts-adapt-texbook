ADAPT-LMS Deadlines and Other Timing

When ADAPT is integrated with an LMS, managing assignment timing can be challenging because both systems have their own independent settings for opening, closing, and late submissions. For students to access ADAPT questions, the assignment windows in both ADAPT and the LMS must be open; if either is closed, access is blocked. Instructors therefore need to choose from three possible modes of operation to coordinate timing between the two systems.

  • Mode 1: LMS is the Gate Keeper
  • Mode 2: ADAPT is the Gate Keeper (recommended)
  • Mode 3: No Gate Keeper (ADAPT and LMS synced)

Mode 1: LMS is the Gate Keeper

In this mode of operation, ADAPT dates are set broadly (e.g., spanning the overall course duration), while Canvas acts as the primary gatekeeper. Instructors define the specific assignment open and close times within the LMS. This setup can work, but it is not suitable if late submissions or penalties are used, since ADAPT does not track those restrictions. For this reason, we generally do not recommend this configuration. However, if late submissions are not permitted, it can be a practical option. The main limitation is that students must be clearly instructed to ignore ADAPT dates and rely exclusively on the LMS for accurate assignment deadlines.

Mode 2: ADAPT  is the Gate Keeper (recommended)

In this mode of operation, the LMS dates are set broadly (e.g., covering the overall course duration), while ADAPT serves as the primary gatekeeper. Instructors configure the specific assignment open and close times directly in ADAPT. This approach is ideal, as ADAPT reliably manages late submissions and penalties. For this reason, we recommend using this timing configuration. The main drawback is that students must be instructed to disregard the LMS dates and rely solely on ADAPT for accurate assignment deadlines.

Mode 3: No Gate Keeper (ADAPT and LMS synced)

In this mode of operation, neither the LMS nor ADAPT functions as the sole gatekeeperâ€”instead, both enforce the same assignment open and close dates. While this approach can work, it requires instructors to duplicate their efforts in both systems. It also creates complications with handling late submissions and applying penalties. For these reasons, we do not recommend this timing configuration.

A better solution is to develop LMS-specific API bridges that automatically synchronize assignment settings between the LMS and ADAPT and this is under development.
