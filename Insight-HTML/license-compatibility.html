License Compatibility

Source: https://commons.libretexts.org/insight/license-compatibility
Last updated at 09/04/2025 01:14 AM by Delmar Larsen

Description: content with different licenses can be "mixed" subject to certain rules.

When using OER and other openly licensed materials, it’s important to understand how Creative Commons licenses affect remixing and adaptation. Some licenses permit modifications and can be combined freely, while others impose restrictions. For instance, resources under a CC BY-ND (‘No Derivatives’) license cannot be changed or integrated with other content, because the license prohibits any alterations. Before interpreting the table above, it’s important to clarify what ‘compatible’ means. In the context of remixing, compatibility refers to whether the licenses of the original resources allow them to be legally combined, modified, or adapted into a new work without violating any license terms. Another way to frame this is to ask: which licenses can be remixed so that the resulting resource can be released under a single, consistent license that is compliant with the licenses of each separate part?

Collections

We formulate the question in this way since one can always combine content from any range of licenses as long as the combined product is considered a collection and does not have a single license to describe it (instead it is a license stack).

Remixes

When creating a remix, an important consideration is whether the combined work can be released under a single license that satisfies the requirements of all the original licenses. This means examining each constituent resource to ensure that any restrictions—such as attribution, share-alike, or no-derivatives clauses—are compatible. Only when a single license can legally cover the entire remix can the new resource be distributed without violating the terms of the original works.

Example

One can remix a CC BY-SA 4.0 resource with another CC BY-SA 4.0 resource and the resulting work can be released under CC BY-SA 4.0, because the licenses are compatible.

Example

If one tries to combine a CC BY-SA 4.0 resource with a CC BY-ND (‘No Derivatives’) 4.0 resource, there is no single license that satisfies the conditions of both licenses. The CC BY-SA 4.0 resource does not allow for any other license (but does allow for derivatives = adaptation) and the CC BY-ND 4.0 license forbids derivatives and that clause cannot be removed.
